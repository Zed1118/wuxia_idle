## 1 · P0 目标与范围

**目标**:在 stage_02_05——
> 玩家看到青衫剑客蓄力 → 窗口内决定「花预算破招」还是「省着挨一下」→ 立刻见回报(破=Boss 踉跄+全队该窗口增伤 / 不破=挨大伤但留内力)→ 托管也能保守地把这场跑完。

## 2 · 核心机制设计

### 2.1 战斗内力 = 每场预算(M1 + R2 + R3)

```text
进场  currentInternalForce = maxInternalForce(maxIf)   ← P0 改:玩家方从 character.internalForce 改为 maxIf
放技  currentInternalForce -= internalForceCost(现状 default_ground_strategy.dart:281)
普攻  cost 0,不回内力;战斗内无任何回复
战后  下一场全新 setup 又满(maxIf)
```
- **R2**:现状玩家进场取 `character.internalForce`(`battle_state.dart:271`)≠ maxIf;敌方已是 `current=max`(`stage_battle_setup.dart:314`)。P0 把玩家也改 maxIf → **两侧对称、预算上限=UI 的 max,干净**。这是 P0 **唯一碰既有平衡的改动**,必须过 balance_simulator + 复查战斗测。
- **R3 有意设计**:伤害公式用 `currentInternalForce`(`default_ground_strategy.dart:354`)→ **花内力同时降低后续伤害**。所以破招的代价 = 少一次技 **+ 持续降输出**,权衡更真。P0 接受此耦合(不拆 combatResource/damageInternalForce);若实测太惩罚,P1 再拆。
- **调参靶子**:maxIf 要让一场 Boss 战约够 **3-5 次关键技**。simulator 验。

### 2.2 Boss 蓄力 + 破招 + 踉跄(P0 唯一全新机制)

```text
青衫剑客被 AI 选中自己的招牌技(chargeSkillId)→ 不立即出伤,进入 charging:
  · chargeTicksRemaining = chargeTicks(默认 3)
  · 顶部蓄力条涨 + "可破招"图标亮;蓄力中 Boss 不出普攻(每 tick 递减)
蓄力满(remaining→0)→ 本次行动放招牌技(大伤害),清蓄力态,招牌技上 CD
蓄力中被 canInterrupt 技命中 → 破招:
  · 取消招牌技,Boss chargingSkill=null,招牌技上 CD(防即时重蓄)
  · Boss staggerTicksRemaining = staggerTicks(默认 2):这几 tick 跳过行动
  · staggered 期间 Boss 防御率 ×(1 - stagger_defense_down)→ 全队该窗口增伤
  · 视觉"破!" + 蓄力条碎
```
- 状态落 `BattleCharacter`(domain 层,strategy 写)——这是战斗领域逻辑,**非**音频那条"presentation 不写 BattleState"红线范畴。
- 只 Boss 配 chargeSkillId,2 名小怪不蓄力。

### 2.3 手动关键技 + targeting + aiUsePolicy(M2 + R1 + R4 + R5)

**现状**:`pendingUltimates: Map<int,SkillDef>`(`battle_state.dart:411`);`requestUltimate` `type!=ultimate` 抛(`default_ground_strategy.dart:118-124`);消费在 `:302-314`;AI 优先级 pending>jointSkill>powerSkill>attack(`battle_ai.dart:39-75`);目标=对面血最低(`_pickTargetId:78-101`)。

**P0 改**:
1. **泛化手动请求**:`requestUltimate` 放宽接受 `powerSkill`(破招技),或加 `requestManualSkill(characterId, skill)` 同写 pending。callsite 少(`battle_screen.dart:473`),直接泛化无需 facade。pending 仍存 `SkillDef`(**不升 PendingManualSkill**,R1 用 targeting 规则替代)。
2. **battle_ai 改 3 处(R4)**——破招技标 `aiUsePolicy=saveForInterrupt`:
   - `_pickSkill` 正常 powerSkill 循环(`:55-64`)**跳过** saveForInterrupt 技;
   - 新增高优先分支:**若对面有敌人 charging 且自己有可用 saveForInterrupt 技 → 用它**(托管保守破招=parity);
   - **targeting 规则(R1)**:选中的技 canInterrupt 且对面有 charging 敌人 → 目标锁定该 charging 敌人(不走"血最低")。手动 pending 与托管自动都受益。
3. **manual_only/normal** 留枚举位,P0 只实装 normal + saveForInterrupt 行为。

## 6 · 托管 parity(M5 + R4)

- 托管靠 `aiUsePolicy=saveForInterrupt`:平时**不乱放**破招技,敌人蓄力时**自动用它破招并锁定蓄力敌人**(§2.3)→ **离线/自动能独立通 stage_02_05**(不假设手操)。
- **手动优势来源**:玩家可(a)卡蓄力快满才破(最大化收益)、(b)破后选在破绽窗口集火、(c)判断"这次不破、省内力+保输出打下阶段"(R3 让省内力同时保伤害)。综合手动比托管多榨 **10-25%**。
