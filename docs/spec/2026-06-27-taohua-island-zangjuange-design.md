# 桃花岛二期：经营据点与藏卷阁联动 · Design

> 日期：2026-06-27
> 阶段：1.0 长线打磨期
> 状态：用户已拍板方向。本文只定设计与拆分边界，不直接实现代码。

## 1. 目标

把桃花岛从“可用的生产卡片列表”升级为玩家重走江湖前的经营据点；把已有战绩册、兵器谱、奇遇录、藏经阁聚合为“藏卷阁”，让档案从单纯收藏变成长期目标与整备线索。

核心循环：

```text
江湖推进 / 周目挑战
→ 藏卷阁暴露缺口与线索
→ 回桃花岛生产、加工、疗养、整备
→ 再出岛挑战高周目 / Boss 异势 / 掉落缺口
```

## 2. 设计原则

- 不做每日任务、每日刷新、订单日课、生产加速、付费感资源闸。
- 桃花岛是低压力后勤据点，不是工厂经营游戏。
- 常规生产链保持 2 层：原料建筑 → 加工建筑 → 现有系统资源。
- 少数终局目标用“岛务工程碑”承接伪 3 层需求，但不形成频繁队列。
- 藏卷阁只给线索，不发日课奖励；线索服务玩家自主选择下一趟江湖。
- 资源、文案、数值继续走 `data/*.yaml`、`UiStrings`、既有 def/config loader，禁止散写。

## 3. 子系统边界

### 3.1 桃花岛经营底座

沿用现有 `source / processor / recipe / cap / upgrade` 模型，扩展到 6-8 个建筑。服务层优先复用：

- `lib/features/taohua_island/application/island_production_service.dart`
- `lib/features/taohua_island/application/island_settle_service.dart`
- `lib/features/taohua_island/application/island_action_service.dart`
- `lib/features/taohua_island/domain/taohua_island_config.dart`

常规建筑建议：

| 建筑 | 类型 | 产线定位 |
|---|---|---|
| 草药圃 | source | 产草药 |
| 木工坊 | source | 产木材 |
| 灵泉 | source | 产灵泉水 |
| 丹房 | processor | 草药/灵泉水 → 疗伤丹、经验丹辅材 |
| 铁匠炉 | processor | 矿石 → 磨剑石、心血结晶辅材 |
| 铸造台 | processor | 木材/矿石 → 开锋辅材、行囊补给 |
| 行商码头 | utility | 低频补缺口，固定货架或低压轮换，不做每日刷新；首批可先做为据点面板，不强行塞入生产 config |

### 3.2 藏卷阁

新增 Hub，不重写现有详情页。首批聚合：

- 战绩册：Boss 记录、周目形态、未破异势。
- 兵器谱：装备获得记录、来源线索、缺口。
- 奇遇录：触发地貌、天气、敌类线索。
- 藏经阁：武学图鉴、残页来源、解锁进度。

藏卷阁的职责是“导航与线索”，不是新奖励系统。

### 3.3 整备建议联动

桃花岛读取藏卷阁线索后显示少量整备建议：

- 缺装备：提示来源章节/周目，并建议准备强化或疗伤资源。
- 缺残页：提示来源塔层/Boss，并建议补足开锋或破招材料。
- Boss 异势未破：提示该 Boss 的周目词条与可准备的后勤资源。

整备建议只显示方向，不自动派任务，不强制跳转。

## 4. 拆分计划

本设计拆为三个独立分支，避免一个会话吞掉所有实现：

| 分支 | 目标 | 可独立验收 |
|---|---|---|
| `codex/taohua-island-phase2-foundation` | 桃花岛建筑扩容、生产链、据点式主屏 | 桃花岛可独立生产/收取/升级新建筑 |
| `codex/zangjuange-hub-clues` | 藏卷阁 Hub + 三类线索 | 可从主菜单进入藏卷阁并查看线索 |
| `codex/island-prep-advice-integration` | 桃花岛读取线索显示整备建议 + 岛务工程碑初版 | 桃花岛能根据档案缺口显示准备建议 |

依赖顺序：

1. 分支 A 先做桃花岛底座。
2. 分支 B 可与 A 并行，但最终合并时需要跟主菜单入口协调。
3. 分支 C 依赖 A+B 合并后的数据接口，不应提前实现生产逻辑。

## 5. 首批不做

- 不做自由摆放建筑。
- 不做装饰加成。
- 不做弟子派遣队列。
- 不做每日订单。
- 不做随机每日行商刷新。
- 不做新的战斗数值倍率系统。
- 不做 PVP 或跨玩家交易。

## 6. 验收标准

- `flutter analyze` 0 issue。
- 相关 feature 单测通过；最后合并批跑全量 `flutter test`。
- 桃花岛离线=在线仍由 `IslandProductionService.settle` 单一函数保证。
- 藏卷阁线索没有任务奖励或日课暗示。
- UI 文案集中到 `UiStrings` / `EnumL10n` 等合法 sink。
- 新增 item/building/recipe 配置有 loader 校验和引用完整性测试。
