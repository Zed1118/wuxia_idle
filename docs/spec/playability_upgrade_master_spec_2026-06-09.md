# 游戏可玩性加强整体方案 · Master Spec

> 日期:2026-06-09
> 目的:给后续 Claude/Codex 实现与 code review 使用的可执行规划文档。
> 状态:方案收束稿。本文不直接改 GDD,后续实施前再拆成 phase spec / plan。
> 上游:2026-06-09 与用户连续头脑风暴,主题从“手动技能战斗”升级为“完整游戏可玩性加强”。

## TL;DR

本方案把当前“自动战斗 + 自动结算”升级为“挂机底盘 + Boss 手动关键技能 + 长期可见养成反馈”的完整游戏可玩性框架。

核心方向:

- 普通战斗、离线结算继续自动,不破坏挂机定位。
- Boss / 爬塔 / 关键战斗加入玩家手动点选全队关键技能。
- 技能有单体、群体、debuff、破招、资源消耗、熟练度成长。
- 所有“越用越强”的系统统一用进度条展示当前阶段、当前效果、下一阶段效果。
- 队伍从 1 人开局逐步成长到 3 人,收徒必须改变战斗方式。
- Boss 不再只是血厚攻击高,每个关键 Boss 必须有机制、阶段、弱点、抗性、掉落和失败复盘。
- 胜利后有英雄镜头与珍稀掉落展示;失败后有短复盘与可跳转调整建议。
- 长期目标通过藏经阁、兵器谱、战绩册、门派谱沉淀,不做日课、红点、体力、登录奖励。

## 一 · 硬约束

所有实现必须继续遵守 GDD 与 AGENTS 红线:

- 不做体力、每日任务、登录奖励、战令、VIP、抽卡、装备分解、强化降级、在线 buff、挂机加速。
- 不让手动收益高到逼玩家长期在线操作。手动优势目标为 10%-25%。
- 离线结算只能走自动托管策略,不能假设完美手操。
- 不引入 Flame 等第三方游戏引擎,战斗表现继续 Flutter Widget + AnimationController。
- 不突破 GDD 数值红线:普通伤害 8000、大招暴击不进十万、玩家血量 20000、内力 15000、装备攻击 2000。
- 低境界不能使用高阶心法、装备或技能。Boss 技能书可提前获得和收藏,但不可绕过三系锁死。
- Dart 不硬编码中文文案与数值。中文 UI 文案走字符串/配置层;数值走 YAML。
- DeepSeek 负责 narrative/lore/events 文案;Mac/Claude 负责 lib、data 根 yaml、test、GDD/spec。

## 二 · 已确认核心决策

### 2.1 战斗操作

- 玩家控制全队关键技能,不做完整微操。
- 普攻、站位、普通循环仍自动。
- Boss 战玩家主要决策:
  - 什么时候释放技能。
  - 用单体还是群体。
  - 用输出、破招、破防、控制、回复还是内伤。
  - 谁来释放关键技能。
- 自动托管可释放技能,但策略保守。

### 2.2 技能分类

技能有两个基础维度:

```text
range: single | group
effect_tags:
  damage
  interrupt
  defense_down
  slow
  seal
  internal_wound
  weaken
  heal
  shield
```

基础规则:

- 单体技能倍率高,主要打 Boss、精英、破招、斩杀。
- 群体技能单体倍率低,主要处理 3v3、小怪、群战波次。
- 大多数 debuff 不叠加,只刷新持续时间。
- `internal_wound` 可最多叠 3 层。

### 2.3 技能资源

技能统一消耗“战斗内力槽”,不新增怒气、行动点、真气等第二资源。

```text
角色总内力 = 成长属性,决定伤害和战斗内力槽上限
战斗内力槽 = 当前战斗可消耗资源
普攻/时间/受击 = 回复少量战斗内力
小技能 = 少量内力 + 短冷却
强技能 = 中量内力 + 长冷却
大招 = 大量内力或达到阈值
```

### 2.4 技能来源

```text
心法 = 决定技能池、流派、修炼度倍率
武学领悟 = 解锁具体技能
Boss 真解/残页 = 稀有招式来源
装备共鸣 = 少量“人剑合一”特殊技
```

Boss 技能书规则:

- 主线 Boss 首通必掉代表性技能真解。
- 爬塔 Boss 概率掉残页,集齐后解锁。
- 重复挑战不重复掉完整真解,可掉残页、熟练度或领悟进度。
- 高阶技能书可收藏,境界/心法不达标时不可装配。

### 2.5 技能成长

技能不做传统 `Lv.1/Lv.2/Lv.3`,改做“招式熟练度”。

| 阶段 | 使用次数建议 | 强化方向 |
|---|---:|---|
| 初识 | 0 | 基础效果 |
| 顺手 | 30 | 冷却略减或消耗略降 |
| 熟练 | 100 | 伤害或命中小幅提升 |
| 精通 | 300 | 核心效果增强 |
| 化境 | 800 | 招式特性、特效或关键小强化 |

综合加成建议上限:

```text
初识 100%
顺手 105%
熟练 112%
精通 120%
化境 130%
```

注意:

- 低阶技能不能靠熟练度无限压制高阶技能。
- 群体技能单体倍率必须低于同阶单体技能。
- 熟练度不能突破 GDD 数值红线。

### 2.6 技能装配

系统层每名角色最多装配:

```text
主修心法技能 x2
辅修心法技能 x1
装备共鸣技能 x1
大招 x1
```

战斗 UI 默认轻量展示:

```text
每名出战角色 1-2 个关键按钮
全队 1 个特殊按钮:大招 / 人剑合一 / 阶段破招
```

队伍人数未满时,技能栏动态收缩。

## 三 · 熟练度与进度展示系统

所有“越用越强”的系统必须统一展示:

```text
当前阶段
当前进度 / 下一阶段阈值
进度条
当前阶段效果
下一阶段效果
进度来源
```

覆盖:

| 系统 | 成长来源 | 展示 |
|---|---|---|
| 技能熟练度 | 技能释放次数 | 当前熟练阶段、当前强化、下一阶段强化 |
| 武器共鸣度 | 战斗次数 / 闭关折算 | 生疏、趁手、默契、心剑通灵 |
| 心法修炼度 | 招式使用 / 闭关修炼 | 9 层修炼度、当前倍率、下一层倍率 |
| Boss 残页领悟 | 残页数量 / 领悟进度 | 已收集残页、解锁条件、掌握效果 |

UI 文案示例(实现时不能硬编码):

```text
裂山斩 · 熟练
进度:186 / 300 次
当前效果:伤害 +12%,破招值 +10%
下一阶段:精通,破招窗口 +0.5 秒
```

设计要求:

- 不能只显示百分比。
- 下一阶段效果要提前可见。
- 视觉语言统一,不要每套系统各画各的。
- 气质像武学谱、兵器谱、藏经阁,不要像手游任务条。

## 四 · 队伍成长与角色定位

### 4.1 队伍人数渐进解锁

不建议开局就是 3 人队。推荐节奏:

| 时间 | 队伍 | 重点 |
|---|---|---|
| 0-15 分钟 | 祖师 1 人 | 1 个手动技能 |
| 15-45 分钟 | 祖师 1 人 | 第 2 技能 + 第一次破招 |
| 45-90 分钟 | 祖师 1 人 | 装备、心法、技能熟练度出现 |
| 90-150 分钟 | 祖师 + 大弟子 | 双人协同 |
| 2.5-4 小时 | 祖师 + 大弟子 + 二弟子 | 完整 3 人队 |

影响:

- GDD §10 新手节奏后续需要同步。
- 早期 Boss 需要支持 1v1 / 2v1。
- 战斗 UI 必须按队伍人数动态适配。

### 4.2 三人默认职责

前期固定倾向,后期自由培养。

| 角色 | 前期定位 | 战斗价值 |
|---|---|---|
| 祖师 | 主控核心 | 爆发、破招、决定战斗节奏 |
| 大弟子 | 稳定前排 | 承伤、破防、保护队伍 |
| 二弟子 | 变化辅助 | 控制、内伤、回复、补足策略 |

后期不锁职业。玩家可通过心法、技能书、装备开锋转型。

## 五 · Build 深度

Build 由四层组成:

```text
流派:刚猛 / 灵巧 / 阴柔
心法:主修 + 辅修,决定技能池
技能:单体/群体/debuff/破招,决定操作方式
装备开锋:攻击/速度/吸血/破甲,补强短板
```

示例 build:

| Build | 强项 | 短板 |
|---|---|---|
| 刚猛破招爆发流 | 高单体、打断 Boss、斩杀 | 冷却长、被克制时不稳定 |
| 灵巧多段暴击流 | 行动快、多段、清小怪 | 单次破招弱、怕内伤 |
| 阴柔内伤控制流 | 持续伤害、虚弱、封招 | 启动慢、爆发弱 |
| 破甲协同流 | 大弟子破防,全队爆发 | 依赖窗口 |
| 吸血续战流 | 稳定、容错高 | 战斗时间长 |
| 速度压制流 | 多次行动、频繁 debuff | 单次伤害低 |

规则:

- 每个 build 必须有强项、短板、Boss 高光场景。
- 同一 Boss 至少支持 2-3 种可行打法。
- 装备开锋不能变成固定答案。
- 心法相生是隐藏彩蛋,不是唯一最优解。
- 伤害最高的 build 不能永远最优。

## 六 · Boss 设计标准

关键 Boss 必须从“数值怪”升级为“机制承载点”。

建议配置字段:

```yaml
boss:
  core_mechanic: ...
  phases:
    - hp_threshold_pct: 0.70
      behavior: ...
  weaknesses:
    - interrupt
    - defense_down
  resistances:
    slow_effectiveness: 0.5
    seal_applies_to: normal_skill_only
  skill_book_drop:
    type: technique_manual
    id: lie_shan_zhan_manual
    first_clear_guaranteed: true
  failure_hints:
    - trigger: killed_by_charge
      hint_key: battle_hint_keep_internal_force_for_interrupt
```

质量门槛:

- 每个主线 Boss 至少 2 阶段。
- 章节最终 Boss 至少 3 阶段。
- 每个 Boss 至少有 1 个弱点和 1 个抗性。
- 每个 Boss 至少服务 1 个系统教学或 build 高光。
- Boss 掉落技能必须反映它在战斗中使用过的招式。
- 失败后必须能给出主要原因和调整方向。

Boss 抗性推荐:

| 效果 | 对 Boss |
|---|---|
| 破防 | 有效,幅度有上限 |
| 内伤 | 有效,最多 3 层 |
| 虚弱 | 有效,但 Boss 大招不吃满额 |
| 减速 | 有效但打折,例如 50% |
| 封招 | 只封普通技能,不封阶段大招 |
| 破招 | 专门处理蓄力/阶段大招 |

## 七 · 战后体验

### 7.1 胜利流程

```text
胜负定格
→ 英雄镜头
→ 珍稀掉落展示
→ 战后复盘
→ 下一步建议
```

珍稀展示触发阈值:

| 类型 | 触发规则 |
|---|---|
| Boss 首胜 | 必触发英雄镜头 |
| 装备 | 利器及以上首次获得触发;重器及以上每次触发 |
| 技能书 | 真解必触发;残页集齐时触发 |
| 材料 | 稀有材料或数量超过阈值触发 |
| 熟练突破 | 可触发短展示,但不抢主镜头 |

展示约束:

- 动画 2-4 秒,可点击跳过。
- 重复同类展示走快速版。
- 普通掉落不强展示。
- 离线结算不逐条播动画,只汇总高价值收获。
- 气质是战利品/兵器谱入册,不是抽卡开箱。

### 7.2 失败复盘

战后复盘三段式:

```text
1 个主要原因
2 条关键数据
2 条调整建议
```

诊断例:

| 失败原因 | 推荐调整 |
|---|---|
| 被 Boss 蓄力大招击倒 | 保留内力,装配破招技能 |
| 被小怪拖死 | 装配群体技能,优先清场 |
| 输出不足超时 | 提升技能熟练度,使用破防技能 |
| 前排太脆 | 强化护具,使用虚弱/回复 |
| 被内伤拖死 | 速杀、回复、虚弱、抗性心法 |

建议应可跳转到相关界面:技能、装备、心法、队伍。

## 八 · 长期目标与收藏系统

四本核心档案:

| 档案 | 内容 | 目的 |
|---|---|---|
| 藏经阁 | 心法、技能书、残页、领悟条件 | 承载技能和武学收集 |
| 兵器谱 | 装备、共鸣、开锋、典故、战绩 | 承载“一柄兵器一段故事” |
| 战绩册 | Boss 首胜、英雄镜头、珍稀掉落、战报摘要 | 承载关键战斗回忆 |
| 门派谱 | 祖师、弟子、传承遗物、飞升传位 | 承载师徒传承 |

长期目标:

- 收齐某流派核心招式。
- 让一件武器达到心剑通灵。
- 集齐爬塔 Boss 残页。
- 完成关键 Boss 首胜战绩。
- 完成门派传承。
- 把一门心法练到极境。

约束:

- 不做每日任务。
- 不做登录奖励。
- 不做红点清单。
- 关键收藏要有保底路径。

## 九 · 探索奇遇与材料经济

### 9.1 奇遇联动

奇遇触发因子:

```text
地图
角色属性:悟性、机缘、流派
战斗行为:技能使用、敌人类型、Boss 战绩
装备经历:共鸣、开锋、曾斩 Boss
失败记录:连续败给同一 Boss
```

示例:

- 古剑冢更容易触发武器共鸣、剑类技能、兵器典故。
- 藏经阁更容易触发心法领悟、技能残页、修炼度提升。
- 悬崖瀑布更容易触发内力突破、刚猛/灵巧悟招。
- 连续败给 Boss 后可触发传闻/师徒对话,给打法线索,不直接送数值。

### 9.2 目标型材料系统

材料必须有明确用途:

| 类型 | 例子 | 用途 |
|---|---|---|
| 普通材料 | 磨剑石 | 装备强化 |
| 保底材料 | 心血结晶 | 强化失败保底 |
| 技能材料 | Boss 残页 | 解锁 Boss 招式 |
| 稀有材料 | 开锋材料/特殊矿石 | 高阶开锋、特殊强化 |
| 事件材料 | 奇遇信物 | 解锁特殊事件或藏经线索 |

规则:

- 关键材料显示目标进度。
- 珍稀材料掉落展示要说明用途。
- 高价值目标必须有保底路径。
- 不做装备分解、每日材料副本、体力限制、复杂货币体系。

## 十 · UI 与表现

### 10.1 信息架构

主结构按玩家意图分区:

```text
江湖推进:
  主线 / Boss / 爬塔 / 闭关地图

门派养成:
  角色 / 心法 / 装备 / 技能 / 熟练度

江湖档案:
  藏经阁 / 兵器谱 / 战绩册 / 门派谱
```

首页只放 1-3 个当前重要行动:

- 昨夜发生的事。
- 当前主线目标。
- 闭关状态。
- 队伍可提升点。
- 下一场 Boss / 爬塔入口。

### 10.2 战斗 UI 层级

```text
上:Boss 状态区
  血量 / 阶段 / 蓄力条 / 弱点图标 / 关键 debuff

中:战斗表现区
  角色站位 / 攻击动画 / 伤害反馈 / 英雄镜头

下:技能操作区
  技能按钮 / 战斗内力槽 / 冷却 / 不可用原因 / 自动托管
```

战斗中只显示影响当前 3 秒决策的信息。

必须展示:

- Boss 蓄力条与可破招窗口。
- 技能冷却与内力。
- 关键 debuff 层数。
- 技能不可用原因。
- 自动/手动状态。

少展示或折叠:

- 详细公式倍率。
- 所有小 buff 列表。
- 普通小怪完整状态面板。
- 非当前决策相关的养成提示。

### 10.3 反馈优先级

| 级别 | 场景 | 反馈 |
|---|---|---|
| S | Boss 破招、大招斩杀、珍稀掉落 | 镜头 + 音效 + 特效 |
| A | 技能释放、暴击、Boss 阶段转换 | 明显动画 + 音效 |
| B | 普攻、debuff 生效、熟练突破 | 轻反馈 |
| C | 普通掉落、小额经验、材料累计 | 列表或轻提示 |

三流派表现:

- 刚猛:轻震屏、红色震波、重击停顿。
- 灵巧:残影、金光、快速连击、暴击闪烁。
- 阴柔:紫雾、内伤数字、延迟爆发、缠绕特效。

可及性:

- 战斗速度:0.75x / 1x / 1.5x / 2x。
- 技能暂停:关闭 / 技能面板减速 / 技能面板暂停。
- Boss 提示:简略 / 标准 / 详细。
- 震屏:关闭 / 轻 / 标准。
- 动画:完整 / 快速 / 重复自动跳过。

排行榜公平性暂不纳入本阶段。

## 十三 · 分阶段实施建议

### P0 · 手动 Boss 战最小闭环

目标:打通“Boss 战可点技能”。

- BattleState 支持战斗内力槽、技能冷却、技能命令。
- BattleStrategy 支持玩家技能 request。
- 技能定义最小 schema。
- Boss 蓄力与破招最小机制。
- 战斗 UI 展示 Boss 蓄力、内力、技能冷却。
- 自动托管保守策略。

验收:

- 1v1 Boss 中玩家可手动释放单体技能和破招技能。
- 自动托管可跑完同一场。
- 离线/自动不假设完美手操。
- §5.4 红线测试仍过。

### P1 · 技能来源与熟练进度

目标:让技能进入养成闭环。

- 心法技能池。
- Boss 真解/残页解锁。
- 技能熟练度计数与阶段效果。
- 统一进度条组件。
- 藏经阁最小入口。

验收:

- 主线 Boss 首通掉技能真解。
- 技能使用次数提升熟练阶段。
- UI 可见当前/下一阶段效果。
- 低境界无法装配高阶技能。

### P2 · 队伍成长与三人协同

目标:从 1 人战斗扩展到 3 人队。

- 队伍人数渐进解锁。
- 2 人/3 人技能栏动态适配。
- 三人默认职责技能模板。
- 至少一个 Boss 支持协同窗口。

验收:

- 开局不展示 3 人队。
- 大弟子/二弟子加入后战斗 UI 自动扩展。
- 三人技能可形成破招→破防→爆发或内伤链路。

### P3 · Boss 标准与战后体验

目标:让关键 Boss 有机制和结果反馈。

- Boss 六字段配置。
- 阶段变化、弱点、抗性。
- 战后复盘三段式。
- 珍稀掉落展示。
- 英雄镜头最小版。

验收:

- 每个改造 Boss 至少 2 阶段、1 弱点、1 抗性。
- 胜利时高价值掉落有特殊展示。
- 失败时给出主要原因和可跳转建议。

### P4 · 长期档案与探索联动

目标:完整游戏长期吸引力。

- 藏经阁、兵器谱、战绩册、门派谱。
- 地图/行为/装备/战绩触发奇遇。
- 目标型材料进度。

验收:

- 玩家可回看 Boss 首胜、关键装备战绩、技能残页进度。
- 奇遇触发能解释其来源条件。
- 材料有明确用途和进度目标。
