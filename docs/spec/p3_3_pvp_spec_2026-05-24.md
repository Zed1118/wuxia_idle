## 2. schema 改动

```dart
// lib/features/pvp/domain/pvp_record.dart(新)
@Collection()
class PvpRecord {
  Id id = Isar.autoIncrement;
  late String matchId;                  // uuid v4
  @Index(composite: [CompositeIndex('timestamp')])
  late int playerId;                    // 本地玩家 character id
  late int opponentSnapshotId;          // 指向 PvpSnapshot.id
  late int leftSnapshotId;              // 玩家阵容快照(冻结)
  late int? winnerId;                   // null = draw
  late int playerEloBefore;
  late int playerEloAfter;
  late int eloDelta;                    // +/- 单场积分变化
  late DateTime timestamp;
}

@Collection()
class PvpSnapshot {                     // 阵容快照(冻结的 3 角色 stat)
  Id id = Isar.autoIncrement;
  late String snapshotJson;             // BattleCharacter[3] JSON 序列化
  late int snapshotElo;                 // 拍快照时段位
  late DateTime takenAt;
}
```

## 4. PvpService + PvpStrategy 行为

- **`PvpStrategy implements BattleStrategy`**(`lib/features/pvp/domain/strategy/pvp_strategy.dart` ~80 行):组合委派 `DefaultGroundStrategy`(零代码重复 · 沿 LightFoot/MassBattle 体例)· `tick` / `runToEnd` / `requestUltimate` 全转发 · ctor 接 `opponentSnapshot: PvpSnapshot` 注入 rightTeam
- **`PvpService.match(playerId)`**(`lib/features/pvp/application/pvp_service.dart`):① 读 player 当前 PvpSnapshot(无则 createSnapshot)② 调 `PvpSyncService.findOpponent(playerElo, window)`(NoopPvpSync 返本地假对手:沿同 ELO 段位生成 mirror team)③ `PvpStrategy(opponentSnapshot).runToEnd` → `BattleResult`④ ELO 算 `delta = K * (actualScore - expectedScore)`,`expectedScore = 1/(1+10^((opp-self)/400))`⑤ 写 `PvpRecord` + 更新 player ELO + 触发 `PvpSyncService.uploadResult(record)`(Noop = 0 网)
- **`PvpSyncService` 抽象**(沿 `LeaderboardSyncService` 体例,`lib/features/pvp/application/pvp_sync_service.dart`):3 method `findOpponent / uploadSnapshot / uploadResult` · `NoopPvpSync` 默认实装(本地 mock 对手生成)· `SupabasePvpSync` Phase 5 替换注入(占位 stub · 真 Supabase 包 1.1+ 上探)
- **离线快照体例**:player snapshot 在 ① PvpScreen 首入时 ② 阵容编辑后 ③ 段位变化时三时机刷新 · `snapshotJson` 序列化 `BattleCharacter` 全 stat(realm/attrs/equipment derived/cultivation/etc)· 复盘可重建战斗
