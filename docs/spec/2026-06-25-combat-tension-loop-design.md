# 战斗张力循环设计（内力弹药 + 境界门槛 + 双层伤势）· design

> 状态：design（用户已确认骨架 → 待 review spec → writing-plans）
> 阶段：1.0 长线打磨期 · 实玩体验深度打磨轴
> 范围：强化两套现有系统（内力 / 境界锁步）+ 一套新系统（per-角色双层伤势）
> 模型建议：实装期 xhigh（动 Isar schema + 战斗结算 + 挂机 tick + 派生属性，跨模块需一次想全）
> **本 spec 只设计，不写代码（用户要求）。**

## 1. 由来 & 红线决策（必读）

用户提出加「体力系统」（挂机获取体力、战斗消耗、没体力不能战斗）。

**体力直接撞项目最硬红线**：§5.1 反主流不做清单第一项就是「体力」（"任何 PR 涉及 → 停"）；连带撞 §5.5 在线=离线、§9 ❌ 清单。体力内核 =「没体力→停手→等恢复/明天再来」= 手游留存/逼氪标志机制，与项目（买断制 / 反主流 / 写实武侠 / 在线=离线）灵魂对立。

Brainstorm 厘清：用户要的不是体力本身，而是三个**设计目标**——① 给推进加自然刹车 ② 给战斗加资源张力 ③ 让挂机产出有去向。用户**未选**「就是想要体力机制本身」。结论：**不做体力，用守红线的写实机制达成这三目标**，用户拍板三套全上。

**用户拍板的边界（不可破）**：
- 伤势两层（小伤连战累积+调息即恢复 / 重伤硬仗后留+疗养）。
- 伤势 per-角色（非全队共享条）。
- 伤势红线铁律：带伤永远能打（伤是 debuff 非闸）/ 疗伤优化非必需 / 无加速疗养（在线=离线）/ 不留存焦虑（不隔夜废角色）。

## 2. 总循环

```
挂机/闭关 ──→ 养内力上限·境界·熟练度(变强) + 疗角色伤势
   │                          │
   │                          ▼
   │              出战(内力满·调息毕) ──→ 闯主线/塔
   │         ┌────────────────┼─────────────────┐
   │         ▼                ▼                 ▼
   │   境界门槛打不过      战斗内内力取舍       连战累积小伤
   │   (推进刹车 ②)       (战斗张力 ①)        (节奏张力 ③轻)
   │                          ▼
   │              硬仗惨胜/败 → per-角色重伤 debuff (③重)
   └──── 带伤硬撑 / 换二弟子上 / 挂机疗养(阵容决策) ◀─┘
```

## 3. 三套机制

### ① 内力弹药（强化现有内力，不新增系统）
- 战斗内内力有限、**不恢复**（已实装）→ 显眼化为「核心弹药」：招式标内力成本、内力见底只能平 A、UI 强调弹药感。
- 出战前内力满（写实「调息毕，提气出战」）。
- 挂机/闭关养**内力上限**（+ 招式熟练度）= 弹药更足 = 变强。
- **守红线**：内力是战斗*内*资源、开战即满 → 非战斗*间*闸，与体力本质不同。
- 改动面：偏配置/表现层 + 派生（内力上限成长）。轻量。

### ② 境界门槛（强化现有锁步，不新增系统）
- 章末 Boss / 塔关键层做成明确**跨阶门槛**（参考 memory：跨 1-2 阶才稳触发战败）。
- 「打不过高阶 → 去挂机/闭关/换装变强 → 再来」= 写实刹车。
- **守红线**：纯实力门槛（§5.3 锁步已在），零资源闸。
- 改动面：偏数值平衡（敌人 scale / Boss 配置），无新系统。轻量。

### ③ 双层 per-角色伤势（唯一新系统，重工程）
- **小伤（疲劳）**：连续作战累积（连打 N 场不调息）→ 小幅减战力；**调息即恢复**（一次挂机 tick / 收功 / 休整）。= 连战 vs 适时调息的节奏张力，不隔夜不留存。
- **重伤（内伤）**：硬仗（Boss / 心魔 / 塔 Boss）惨胜或战败后，该弟子留 debuff（内力上限 / 攻击打折）；靠挂机/闭关**真实时间**疗养。
- **红线铁律**（§1 已焊死）：带伤永远能打、疗伤优化非必需、无加速疗养、不留存焦虑。
- per-角色 → 重伤弟子：带伤硬撑 / 换二弟子上 / 挂机疗养 → 接已实装的 3v3 阵容 + 二弟子控制（第七阶段）。
- 改动面：Isar schema +字段（saveVer bump）+ 战斗结算判定 + 挂机/闭关 tick 恢复 + 派生属性烘焙减 debuff + UI 显伤势。**本方案唯一重工程。**

## 4. 守红线检查表

| 红线 | 本方案如何守 |
|---|---|
| §5.1 无体力 | 伤势是 debuff 不锁战斗；内力开战即满；全程无「没 X 不能战斗」闸 |
| §5.5 在线=离线 | 疗养/调息 = 真实时间挂机推进，无加速疗养/快进券 |
| 无留存焦虑 | 不疗也能推进（带伤硬刚）；不隔夜废角色；非「每天回来疗伤」 |
| §5.3 锁步 | 境界门槛复用现有锁步，未破 |
| 不硬编码 | 所有阈值/幅度/时长进 numbers.yaml |

## 5. 数据 / 状态变更（实装时）

- `Character` 加 per-角色伤势字段：轻伤累积值 + 重伤 debuff 状态 + 疗养进度 → Isar schema +字段（saveVer bump，迁移旧档默认无伤）。
- 挂机/闭关 tick：顺带恢复轻伤（即时）+ 推进重伤疗养（按真实时长）。
- 战斗结算：连战计数 → 轻伤累积；硬仗结果（惨胜/战败判定）→ 重伤判定。
- 战斗派生属性（derived_stats / BattleCharacter 烘焙）：减去伤势 debuff。
- UI：角色面板 / 出战编成显伤势状态 + 疗养进度。

## 7. 分期建议（实装顺序，writing-plans 细化）

1. **轻量先行**：① 内力弹药显眼化 + ② 境界门槛——纯强化现有、无 schema 改，快速看「张力/刹车」感。
2. **重工程**：③ 双层伤势——新 schema + 战斗结算 + tick + 派生 + UI，单独一期 TDD。
3. 三套上齐后真机/实玩校 balance（§6 参数）。

（用户已拍「三套都上」；分期是实装节奏，非砍 scope。）

## 8. 范围外

- 不做体力 / 任何「没 X 不能战斗」闸。
- 不做加速疗养 / 快进券 / 留存机制。
- 伤势不锁战斗、不隔夜废角色。
- balance 数值本 spec 不定（实装期 + 真机校）。

### 已知设计取舍（**故意如此，非 bug，勿"修"**）

1. **塔 Boss 战败不触发重伤**：塔 defeat 路径不调 `BattleResolutionService.resolve`（只 recordDefeat），故塔 Boss 战败无重伤（主线 Boss 战败有）。**用户 2026-06-25 拍板不接**——塔可重复挑战，每败重伤恐成「越败越弱」挫败螺旋。塔 Boss **胜利**路径仍接 isHardFight（惨胜低血重伤照常）。
2. **Ch1 末 Boss 宽松**（stage_01_05 弱 build 95% 胜率）：**用户 2026-06-25 拍板接受**——Ch1=学武出山开篇章，Boss 好打降低上手门槛，写实武侠开篇先尝甜头；中后期门槛 + 伤势已提供刹车。**勿擅自收紧 Ch1**。
3. **非 Boss 关战败不累轻伤**：走既有「免费重试无惩罚」分支（不调 resolve），与连战轻伤字面略异但符合既有设计。
