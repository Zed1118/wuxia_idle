# Codex 视觉验收派单 · 半手动战斗 P0 步骤3c 单步 UI

**commit：** fa729bcc · **日期：** 2026-06-13
**验收包：** `build/macos/Build/Products/Debug/wuxia_idle.app`（`./tool/build_acceptance.sh` 预编，VISUAL_ROUTE=hub）
**路由：** hub 总入口 → 点 **`battle_manual_step`**（半手动单步 UI · scenarioChargeBreak 满配主控 + seed=3 确定性）

⚠️ 本路由是**交互验收**（非静态截帧）：进屏后需点击驱动，不是开屏即终态。
分辨率固定 **1280×720**（沿派单惯例，避免窄屏裁切）。

## 验收点（逐项截图 + PASS/FAIL）

1. **入口/初始态**：进屏底部指令台右侧应是实心绛金「**下一步**」按钮（非「快进」）。
   开屏不自动播（画面冻结在 seed 态，无人出手）。
2. **行动顺序条**：点 1~2 次「下一步」推进到 tick 边界填队列后，顶部出现
   「**本回合行动顺序：甲 ▸ 乙 ▸ …**」条，队首角色高亮（绛金描边）。
   截图验：顺序条存在、角色名可读、队首高亮明显。
3. **强制停顿逐步**：继续点「下一步」，每点一次只结算一个 actor（飘字/弹道逐个出），
   行动顺序条随队列缩短、队首前移；队列空后再点 = 新回合边界。验「一步一 actor」节奏。
4. **目标 picker（立即弹）**：选主控（左下头像），点其一个**单体技**（强力/破招/大招）。
   应**立即弹**「选择目标」弹窗，列存活敌人（带血量），无死亡敌人。
   截图验：弹窗水墨样式、敌人项可点、标题「选择目标」。
5. **选目标下发**：picker 里点一名敌人 → 弹窗关闭，该技能按钮盖「待发」印。
   再点「下一步」推进到该 actor 行动时，应攻击所选目标（飘字落在该敌身上）。

## 关注的视觉质量

- 「下一步」按钮与快进区别明显（实心 vs 描边），点击区够大（96×52）。
- 行动顺序条不与顶部危险条（敌蓄力时）打架/重叠，两条都在时排版正常。
- picker 弹窗在 1280×720 居中、不溢出、底色 panel 与战场背景对比够。

## 已知非 bug

- seed=3 是固定确定性场景；蓄力敌人在场时危险条 + 行动顺序条同屏属正常。
- 鼠标合成若不稳（历史 charge_break 验收遇过），改人工点击；FAIL 前先确认是合成问题非 UI 问题。
