## 本批已选待办

### A. 高优先级低冲突表现层

1. **主菜单状态摘要二期**
   - 目标:主菜单显示 3-5 个最重要状态,例如闭关中、桃花岛可收、伤势待处理、主线卡点、可突破。
   - 边界:只读摘要,不改变任何收益、门槛、结算。
   - 适合首批启动:是。入口集中,收益明显。

2. **设置页二期**
   - 目标:集中整理音量分组、战斗速度、文字密度、窗口/全屏、减少闪烁等舒适性选项。
   - 边界:不新增外部服务,不改核心结算。
   - 适合首批启动:是。与其他玩法任务冲突低。

3. **音频专属素材补齐**
   - 目标:替换当前转用的 battleUlt / battleChargeStart 素材,提升大招与蓄力预警辨识度。
   - 边界:只换素材与枚举/manifest 接线,不改战斗逻辑。
   - 适合首批启动:是,但依赖素材生成/筛选。

### B. 中优先级体验增强

4. **资源总览页**
   - 目标:汇总银两、磨剑石、心血结晶、桃花岛产物、丹药、秘籍等库存与主要用途。
   - 边界:只读经营面板,不新增资源、不改经济。
   - 适合首批启动:是。可复用材料来源/用途提示底座。

5. **桃花岛生产队列可读化**
   - 目标:每个建筑显示当前配方、剩余时间、满仓时间、产物去向。
   - 边界:不改产量、不改离线规则、不加加速。
   - 适合首批启动:是,但需避开二梯队若正在改桃花岛相关文件。

6. **战斗 UI 术语统一整理**
   - 目标:统一蓄力、破招、真气、玉体、相位、重伤等术语的显示样式和解释入口。
   - 边界:不做教程弹窗,不改战斗公式。
   - 适合首批启动:可选。影响面跨战斗/情报/战报,需二梯队稳定后再摸排。

### C. 较大 UI 结构改造

7. **关卡列表章节轴重做**
   - 目标:把纯列表改成更有章节节奏的卷轴/路标式结构。
   - 边界:不做地图地域化、不加新收益。
   - 启动建议:二梯队之后第二批。关卡行近期改动多,冲突概率高。

8. **技能详情页表现层升级**
   - 目标:招式详情统一展示流派、破招属性、熟练收益、典型用途。
   - 边界:不改技能数值、不改熟练度公式。
   - 启动建议:可与术语统一配套,但不要和战斗 UI 术语同分支硬绑。

9. **装备典故阅读体验升级**
   - 目标:装备详情里的典故升级为更像器物志的阅读层,支持多段展开、来源与持有人记忆。
   - 边界:中文文案继续走 `data/lore/` 或合法集中层,不在 Dart presentation 散写。
   - 启动建议:中后批。需先确认现有 lore 数据结构能否承载“持有人记忆”。

10. **桃花岛场景化主屏**
    - 目标:把卡片列表升级成可点建筑的场景式界面。
    - 边界:服务层、数值、离线结算不动;素材未齐时先做布局底座和热区架构。
    - 启动建议:较后。UI 改造大,最好等桃花岛生产可读化稳定后做。

### D. 进度/存档侧

11. **多存档体验二期**
    - 目标:存档名称、进度摘要、最近游玩标识、删除保护。
    - 边界:若需 schema/saveVersion,必须单独列高风险计划;优先评估能否用既有字段派生。
    - 启动建议:单独开,不与 UI 大改同批。

12. **新档目标引导二期**
    - 目标:不做教程弹窗,只用主界面/关卡入口轻提示说明当前自然目标。
    - 边界:不做日课、不做强制任务、不加奖励。
    - 启动建议:依赖主菜单状态摘要二期和关卡轴稳定后再做,否则容易重复。

13. **开锋槽 3 专属技内容化**
    - 目标:给装备第三槽补专属技能候选、配置与 UI 空状态。
    - 边界:守三系锁死、数值红线,不把低境界装备变成越阶通行证。
    - 启动建议:单独高风险玩法分支。需要先写 design/spec,再落数据与 UI。
