# 商店货架分层展示二期

## 目标

- 将江湖商店固定货架按购买意图分层展示，便于快速扫读。
- 每个货架组展示标题、简短说明、数量摘要。
- 每件货物继续沿用现有卡片风格，并显示当前库存已有量。

## 边界

- 只做展示增强，不改 `shop.yaml` 库存、价格、购买、刷新、掉落、产出逻辑。
- 不做装备目标追踪、部位缺口提醒、材料替代路径、碎片来源聚合。
- 不修改 `docs/spec/playability_phase2_backlog.md`。
- 新增 UI 文案只放入 `lib/shared/strings.dart`。

## 实施步骤

1. 读取现有商店商品、需求提示与库存 provider。
2. 在 `ShopScreen` 中派生用途货架组：修行补给、兵刃强化、开锋整备、疗伤补给、桃花岛营造、武学秘卷、行囊常备。
3. 将原分类面板替换为用途组面板，保留现有筛选、购买按钮、价格状态和需求提示。
4. 在货物卡片状态区显示当前库存已有量。
5. 更新商店 widget tests 覆盖分组标题、组说明、库存显示和既有筛选行为。
6. 运行商店相关 targeted tests 与 `flutter analyze`。

## 验收标准

- 生产入口 `ShopScreen` 使用真实 `shopItemListProvider`、`allInventoryItemsProvider`、现有 `ShopNeedHintService` 派生展示。
- 货架按购买意图分组，标题、说明、数量摘要可见。
- 每张货物卡显示当前库存数量；无库存时也显示 `已有 ×0`。
- 购买流程仍调用 `ShopService.purchase`，没有新增或修改定价、库存、刷新、产出逻辑。
- Dart 调用点不新增散落中文文案；新增文案集中在 `UiStrings`。
- 测试覆盖：`test/features/shop/*` 通过。
- 静态检查：`flutter analyze` 通过。
