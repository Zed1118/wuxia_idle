# 战斗命中演出分级 · design

> 状态：design（用户已逐屏拍板骨架/特写门槛/完整阶梯 → 待 review spec → writing-plans）
> 阶段：1.0 长线打磨期 · 战斗爽感主旋律（表现层，不走数值膨胀）
> 范围：纯表现层。把现有「一刀切」的命中演出理成一致的等级阶梯 + 新增命中特写镜头。**零 schema 改动、不碰战斗结算/收益。**
> 模型建议：实装期 high（表现层多文件改 + 配置层，但无 schema/算法风险）。
> 关联：① 节奏校值（绝对 ms/px 真机调）与「内力当平衡问题修」是同期 backlog 的姐妹项，本 spec 不含；本 spec 只定阶梯结构与相对递进。

## 1. 由来 & 红线

用户提「让战斗更有爽感」。盘点现状（本会话 Explore 实测）：打击感三档 `ImpactTier{light/medium/heavy}` 已控顿帧/屏震/屏闪/飘字/音效，但**题字系统一刀切**（大招/破招/会心/转阶段全用同一套 56px/同色/同时长），且**无命中特写镜头**。即「大招暴击」和「普通破招」演出长一个样，峰值时刻没被「拍下来」。

**红线（不可破）**：
- 纯表现层，不改伤害/结算/掉落/收益（守 §5.5 在线=离线）。
- 不硬编码：所有演出参数进 `numbers.yaml` 新 `presentation.hit_tier` 段（守 §5.6）。
- 不数值膨胀：爽感走表现层，不靠加伤害/抽卡（守战斗爽感主旋律 / §5.4 软红线）。
- 挂机不被打断：特写在 快进/扫荡/连播/拖招 模式抑制（与现有屏震一致）。

## 2. 设计目标

1. 命中等级在演出强度上**逐级、一致地**拉开（普通 < 强力 < 大招 < 峰值）。
2. 峰值一击（大招暴击 / 击杀）被**镜头特写**强调，稀缺即爽感。
3. 暴击/会心/破招作为**修饰**叠加独立签名，任意基础档上都能触发。

## 3. 真相源：HitTier 由现有字段派生（零 schema 改动）

用户拍板**基础档 × 修饰叠加**（非线性拍平枚举）。所有判定字段已在 `BattleAction` / `AttackResult`，表现层读组合即可：

- **基础档** = `SkillType`（`core/domain/enums.dart` SkillType：normalAttack / powerSkill / ultimate·jointSkill）→ 复用现有 `ImpactTier{light/medium/heavy}`（`impact_profile.dart` `impactProfileFor()`）。
- **修饰层** = `AttackResult.isCritical`（暴击）/ `BattleAction.weaknessHit`（会心）/ `BattleAction.interrupted`（破招）/ 击杀（目标 HP 归零）→ 叠加签名。
- 派生集中到单一 helper（新 `hitTierFor(action) → HitPresentationSpec`），表现层各 overlay 从它读，不各自散判。
- **双方生效**：分级对玩家命中与敌方命中均适用；题字配色沿用现有约定（玩家暖金 / 敌方绛红 `WuxiaColors.gangMeng`）。特写对双方峰值均触发（玩家秒杀敌=爽 / 敌秒杀玩家=惨烈张力，同为峰值时刻）；若真机觉得敌方击杀特写观感不佳，由 config 开关单独关敌方侧。

## 4. 演出预算阶梯

绝对值（标"校"）随 ① 节奏校值真机调；本 spec 定结构与相对递进。括号标"有"=复用现有、"新"=本次新增。

| 命中 | 题字 | 顿帧 | 屏震 | 屏闪 | 特写 | 音效 |
|---|---|---|---|---|---|---|
| 普通命中 | —（有） | 60校（有） | 3px（有） | 0.12 白（有） | — | 平A（有） |
| 强力技 | 单字72 斩/断（有） | 90校（有） | 6px（有） | 0.20（有） | — | —（有） |
| 大招 | 大题56 暖金（有） | 120校（有） | 10px（有） | 0.30（有） | — | 大招音（有） |
| **大招暴击** | **特大68+辉光（新）** | **200校（新）** | 10px | 0.30 绛红（有） | **✓定格（新）** | 暴击+大招（有） |
| **击杀一击** | 承接基础档 **+辉光收束（新）** | **200校（新）** | 承接 | 承接 | **✓定格（新）** | 承接+击杀音（新 hook） |

> 现有 `ImpactTier` 控的维度（顿帧 60/90/120、屏震 3/6/10、屏闪 0.12/0.20/0.30）保持，对齐为同一阶梯的基础三档。新增集中在题字分级 + 峰值特写 + 峰值加长顿帧。

## 5. 修饰层签名（叠在任意基础档上）

- **暴击**（isCritical）：飘字×1.5 红（有，`damage_popup.dart` PopupType.critical）+ 屏闪转绛红（有）+ **题字辉光（新）**。
- **会心**（weaknessHit）：弹「会心」glyph（有，`impact_glyph_overlay.dart`）。
- **破招**（interrupted）：弹「破!」题字（有）+ **顿帧升一档（新）**（破招是反制高光时刻，值得比基础档更停一拍）。

## 6. 命中特写镜头（本次最大新增）

用户拍板**只给绝对峰值**：仅 **大招暴击** + **击杀一击**（最后一击/Boss 终结）触发。前台普通战 0–1 次、Boss 战 1–2 次，稀缺保爽感。

- **效果**：命中帧镜头朝命中点轻推 + 缩放 ~1.1× → 定格 150–250ms（校）→ 弹回正常视角。总时长 ≤0.4s，一闪而过的「这一下好猛」，非过场动画。
- **实现取向**：复用现有屏震的 transform 通道（`battle_screen.dart` 战斗区 Transform），加缩放 + 定格分量；不引入新相机系统、不改布局。题字辉光叠加在定格窗口内。
- **抑制门控**：快进 / 扫荡 / 连播 / 拖招模式不触发（沿用现有屏震的 `非快进/拖招` 判定）。
- **击杀判定**：本次命中使目标 currentHp 归零（表现层读结算后状态，不改结算）。

## 7. 配置

新 `numbers.yaml presentation.hit_tier` 段，集中所有可调参数（题字字号/辉光强度、各档顿帧、特写缩放/定格 ms、屏震屏闪沿用现值的引用）。`AnimationNumbers.defaults` 同步。校值时只动 yaml，不动代码（与 ① 节奏校值同一调参面）。

## 8. 不做（YAGNI）

- 不做所有大招/所有时刻的特写（仅峰值，用户拍板）。
- 不做 slow-mo 全局时间缩放 / 连击数字堆叠 / 弹道笔触分级（本轮排除，未来按需再开）。
- 不动结算、不加伤害、不改掉落。

## 9. 验收

- 代码层：HitTier 派生 helper 单测（各字段组合 → 正确 spec）；演出参数全来自 numbers.yaml（grep 无散写常量）；全量 analyze 0 / test 绿。
- 表现层：真机 `flutter run -d macos` 目检——四档命中演出可辨、峰值特写只在大招暴击/击杀触发、快进/扫荡无特写（与 ① 节奏校值同一次真机 pass 一起校绝对值）。
- 红线：结算/收益零改动（diff 不含 damage_calculator/battle 结算逻辑）。
