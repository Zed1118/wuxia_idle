## 二 · Batch A 架构发现(damage_calculator vs default_ground_strategy)

**关键非显然发现**:`DamageCalculator.calculate`(`damage_calculator.dart`)的 `AttackContext.attacker` 是 `Character` 类型(Isar 实体 · phase1 公式参考实现),**不参与实际战斗**。
战斗实际走 `DefaultGroundStrategy._calculateInBattle`(`default_ground_strategy.dart:343`),`attacker` 是 `BattleCharacter`(view layer 缓存快照)。

→ attackPowerMultiplier 字段**只能加在 BattleCharacter 上**,damage_calculator 改动会引 analyze error(Character 类无此字段)。Phase 0 grep 漏看(只 grep `damage_calculator` 文件名,未 grep `_calculateInBattle` caller),工作中发现回滚 + 改正确路径。

→ 沉淀 memory:**Phase 0 不仅 grep 模块路径,还要 grep caller 路径**(memory `feedback_phase0_grep_two_axes` 扩 caller 维度,留下批 sink)。
