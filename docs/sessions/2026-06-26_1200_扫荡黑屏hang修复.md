- **症状**：进主线一键扫荡卡「连播中 1/5」黑屏无限转圈，停止无效。
- **根因（证伪交接疑点 + 系统化调试定真）**：sweep 自实装从未真机玩。`battleProvider` 是 **autoDispose**（生成码 `isAutoDispose:true` 实证）。SweepScreen 在 `_preparing` spinner 期间不挂 BattleScreen → 无 watcher → `startBattle` 注入队伍后被 autoDispose 回收重置回空团 → 后续挂出的 BattleScreen 拿空团 → 落 BattleScreen 空队伍黑屏 placeholder；且迟挂载错过 `startBattle` 的 `empty→非空` listen 边沿 → timer 永不起。交接疑的 `battle_resolution.resolve` 改动**已证伪**（与本 bug 无关，bug 早于该改动）。对照工作模式 `_StageBattleHost`（stage_entry_flow）是「先挂屏空团、后 postFrame startBattle」靠边沿起 timer，扫荡反序故坏。
- **修复（两处互补·均 opt-in 零回归）**：
  ① `SweepScreen.initState` 加 `ref.listenManual(battleProvider, (_, _) {})` 跨本屏生命周期保活 autoDispose provider，逐关注入不被回收。
  ② `BattleScreen` 加 `autoStartOnMount`（默认 false，保持现有「预填战斗后冻结」契约·零回归；SweepScreen 传 true）：挂载到已就绪活跃战斗且无 timer 时兜底自启，补迟挂载错过的边沿。
- **停止按钮**：现于关边界（`recordVictory` 内 `_stopRequested`）正常生效；原「无效」纯是战斗冻结、`_onVictory` 永不触发所致。
