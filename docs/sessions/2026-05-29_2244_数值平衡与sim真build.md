## 重要决策

- **决定 sim 去 +1 改 on-level 诚实基线，因为** `playerTier=requiredRealm+1` 是旧假 `_synthPlayer` 时代的补偿 hack；真 build 下玩家凭空 +1 阶 vs 同阶敌人，把后段全冲 trivial，制造「全程过低」假象。on-level = 玩家恰在 required 阶 = 诚实下限读数（过度练级只会更易）。**此修正挡下了一次错误重平衡**（原 #2「抬全程敌人数值」基于 confounded sim）。
- **决定 B2 低 tier EXP 回 ×1.0 而非抬阈值，因为** finding 纠正后真实影响是「挂机优先玩家早期战斗碾压」而非跳内容；回原值精准只削低 tier 挂机、不动实战/主线，根因A 中期喂成长初衷在 erLiu+ 保留。
- **决定 balance build 模型用「活跃玩家」单档起步再扩 floor+ceiling，因为** 单 build 偏 ceiling 会高估难度；bracket 隔离配装轴才能区分「中位玩家是否卡」vs「满配是否碾压」。
