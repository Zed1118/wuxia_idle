## 踩坑提醒

- **EnterWorktree 默认 `baseRef: fresh` 基于 origin 非本地 HEAD**（已记 memory `feedback_bg_worktree_baseref_fresh_diverge`）：未 push 的本地 commit 不在新 worktree → ff-merge 到本地 main 分叉。修法：worktree 提交后 `git rebase main` 再 ff-merge（文件不重叠则零冲突）；或开 worktree 前网络通时先 push。本会话 redline 批踩了一次（idle commit 未 push，worktree 缺该文件，全量少 1 测）。
- **fresh worktree 缺 `.g.dart`（gitignored）+ `libisar.dylib`**：跑 test 前从主仓拷全部 `.g.dart`（`find lib test -name '*.g.dart'`）+ 完整 dylib（比对字节数 2187120，截断会致批量 setUpAll dlopen 失败）。本会话两次 worktree 用此法跳过 build_runner。若改动新增 @collection/@riverpod 才需真跑 build_runner（本会话 RedLinesConfig 是纯类，拷 .g.dart 即可）。

- **`flutter test`（无参）输出重定向到文件时进度行 `\r` 压缩**：`grep -c <文件名>` 可能 0 匹配≠没跑。可靠核验用单独跑该 test 文件 + 确认文件 `ls` 存在。本会话误判 idle_economy「没跑」，实为 worktree 缺文件（baseRef 坑），单独跑才暴露真因。
