## 踩坑提醒

- **SSH 嵌套 PowerShell 引号**：`'10' 不是内部或外部命令` 是 PowerShell 解析失败。`git commit -m "..." -m "..."` 多 -m 在 SSH 透传时容易解析错位，**用单 -m + 简短中文 message**（避免 here-doc 多行）
- **file mode noise**：28 处 `.nightshift/*.sh` + `tools/pen_screen/pen_screen.sh` 在 Windows 上 `100755 → 100644` 自动 dirty，**`git config core.filemode false` Pen 本地一次性解决**（不影响仓库）
- **PowerShell GBK → SSH → Mac 终端 UTF-8 多层编码**：SSH 输出看到中文乱码 `招�?` 不等于文件真有乱码。验证必须 `[Console]::OutputEncoding = [System.Text.Encoding]::UTF8` + `Get-Content -Encoding UTF8`
- **flutter test `+N ~M` 中 `~M` 是 skipped 计数（不是 retry/flaky）**：3 个 session doc 历史描述「leaderboardSync 偶发 flaky · Isar 初始化时序」全部错诊断。再遇 flaky 描述必须 git grep 实测 + 单跑 5 次确认

- **PowerShell regex `?` 是「前字符可选 0-1 次」**：写在乱码扫描里会全行命中空串。乱码扫描用 `[�￾﻿]` 等具体码点，或 grep -P
