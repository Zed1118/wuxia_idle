## 2. 当前 UI 总体判断

项目 UI 已经从“可运行的数据面板”推进到“核心玩法可见、主要系统有水墨包装”的阶段。

已经明显改善的部分：

- 战斗单位从小圆头像 / 首字占位，升级为可辨认的敌我头像、弹道、受击反馈、折叠日志与胜负 vignette。
- 角色面板已具备档案头、画像签、装备外观、主修心法视觉区、心魔成长瓶颈提示。
- 仓库与装备详情已从列表 / 白盒展示，改向装备柜、宣纸面板、ItemSlot、典故卷轴。
- 主菜单已从长列表按钮改成修行 / 演武 / 江湖三栏门面。
- UI-kit v1 已落地，后续页面有统一组件基础。

仍未完成的方向：

- 世界层空间感还不足：主线、爬塔、闭关仍未完全成为“地图 / 路径 / 场所”。
- 成长仪式还未形成统一模板：部分事件有强化，但未覆盖所有关键成长事件。
- 心法与三流派关系仍主要靠文字，缺少关系图 / 相生结构化可视化。
- 当前 UI 分支与未提交爬塔改动需要收口，否则规划进展与 `main` 状态会继续分叉。

## 5. 建议执行顺序

建议按以下顺序开展：

1. **T1 当前 UI 分支收口**  
   先让分支成果进入稳定主线，避免后续工作建立在漂移状态上。

2. **T2 爬塔 UI 未提交改动收口**  
   当前已有代码改动，应尽快决定保留、完成或撤出到独立任务。

3. **T5 闭关地图化**  
   P0 核心屏已补，闭关是世界感缺口里最明确的一块。

4. **T3 / T4 主线与爬塔完整空间化**  
   第一切片已有基础，继续推进成完整路线图 / 塔身图。

5. **T6 成长仪式模板**  
   先统一突破、离线结算、Boss 首胜三类高价值反馈。

6. **T7 心法与流派可视化**  
   这是系统理解层的增强，适合在主要页面包装稳定后做。

7. **T8 系统入口状态提示**  
   属于主菜单二次 polish，可与其他工作穿插。

8. **T9 最终截图包**  
   放在本轮 UI 工作完成后统一收口。
