## 0. 设计原则

### 0.1 写实武侠基调（GDD §1.2）

- 写实而非二次元，沉郁而非鲜艳
- 参考意象：水墨、宣纸、竹影、雨夜、青衫、断剑、孤灯
- 色彩克制，UI 不堆砌特效（区别于刚猛流派暴击/克制时的视觉强调）

### 0.2 反主流交互原则（GDD §2）

- **不堆红点**：未读 GameEvent 用淡淡的金色"·"提示，不用红色感叹号
- **不弹任务列表**：上线第一屏是叙事摘要而非「今日任务 5/10」
- **未解锁系统隐藏而非灰掉**：避免认知负担与"我什么时候能玩这个"焦虑
- **不做留存焦虑通知**：桌面通知极简、可关、默认温和

### 0.3 解锁哲学（GDD §10.3）

> "让玩家先感受问题，再给答案。"

- 所有进阶系统**先在战斗/剧情中遇到现象**，再揭晓机制
- 第一小时所有战斗轻松取胜（让玩家爽起来）
- 二周目可跳过引导直接开档

### 0.4 信息密度三档约定

| 档 | 用法 | 示例 |
|---|------|------|
| **极简** | 上线第一屏、主菜单 | 仅 1 屏关键信息 + 1-2 个操作 |
| **标准** | 门派概览、战斗 | 5-8 个操作入口，分组清晰 |
| **密集** | 装备/心法/角色详情 | 数值表 + 操作按钮，专为 build 党设计 |

## 3. 重点：「昨晚发生的事」上线第一屏（A3）

### 3.1 设计意图（GDD §9.2）

> 让"打开游戏"这件事本身具有仪式感——不是看到一堆红点要清，而是看到一段故事的延续。

这不是任务列表，是**叙事摘要**。玩家关掉游戏后真实时间过去了几小时，回来要看到的是「在这段时间里，门派里发生了什么」，而不是「你欠系统多少操作」。

### 3.2 视觉设计

║                       昨夜，门中事                            ║

║                                                                ║

**视觉规范**：
- 背景：暗色宣纸纹理（接近 `#1a1612`），不是纯黑
- 文字：金色（`#c9a961`）+ 衬线字体（思源宋体或类似），略带毛笔感
- 行距：1.8-2.0（呼吸感）
- 渐入动画：每段文字延迟 200ms 浅淡入，让人有"读"的节奏
- 不要任何图标、emoji、彩色 badge
- 整屏不超过 7 段（多了折叠"还有 N 件事"）

### 3.3 内容来源（数据层）

来自三个数据源的合并按时间倒序：

| 来源 | 表 | 字段 |
|------|-----|------|
| 已完成未领取的闭关 | `RetreatSession` (schema §4.7) | `expectedEndAt < now AND isClaimed == false` |
| 离线期间触发的奇遇 | `AdventureRecord` (schema §4.6) | `triggeredAt > saveData.lastOnlineAt` |
| 各种事件流（突破、共鸣、领悟、击败 Boss 等） | `GameEvent` (schema §4.9) | `occurredAt > saveData.lastOnlineAt AND isRead == false` |

每条记录用 `GameEventType` 枚举映射到一个文案模板（DeepSeek 维护，本阶段先用占位）：

```
GameEventType.retreatCompleted     → "卯时，山林闭关结束。带回经验若干、磨剑石数枚。"
GameEventType.skillEnlightened     → "子时初刻，二弟子在悬崖瀑布闭关，听到瀑声中藏着剑意。一招"听水裂石"自掌心生出。"
GameEventType.resonanceUpgraded    → "寅时三刻，大弟子的青锋剑在第二百次出鞘后，剑身微微颤鸣。三月相伴，已是默契。"
GameEventType.adventureTriggered   → "丑时，祖师独自下山。归来时眉间多了一缕剑气。"
GameEventType.bossDefeated         → "亥时末，大弟子在第十二层险胜鬼影刀客，剑上添了一段新典故。"
GameEventType.realmBreakthrough    → "辰时，二弟子破入二流·入门。门内今日多了一壶酒。"
```

**首次开档跳过**：检查 `SaveData.isOnboardingCompleted == false` 或 GameEvent 表为空 → 直接跳到 A4 门派概览，不显示本屏。

### 3.4 主要操作

| 操作 | 行为 |
|------|------|
| **一并领下** | 把所有 RetreatSession 的奖励发到背包 + 标记 GameEvent 已读 + 自动跳到 A4 门派概览 |
| **跳过** | 不领奖励（保留在 A4 顶部"未领取奖励"角标），直接跳 A4 |
| **逐条点击** | 点某条文字进入对应的详情（如点闭关那条 → 闭关结算面板查看具体掉落） |

**30 秒原则**：
- "一并领下"按钮文案明确「30 秒读完」是承诺，对应 GDD §9.2 的「快速领取按钮」
- 实测：从启动到领完所有奖励再回到主页，**不超过 30 秒**（含动画）

### 3.5 边界场景

| 场景 | 行为 |
|------|------|
| 距离上次上线不到 5 分钟（短回归） | 跳过本屏，直接 A4，避免"刚出去抽烟回来还要看一遍" |
| 没有任何待领取奖励 / 事件 | 显示一句轻量话（如「门内一夜清平。」），3 秒后自动跳 A4 |
| 闭关还未结束（玩家提前回来） | 不在本屏列出，留给闭关界面处理 |
| 事件超过 7 条 | 显示前 7 条 + 折叠按钮「还有 X 件事」，点开看全部 |

### 3.6 新手引导锚点

| 触发时机 | 引导内容 |
|---------|---------|
| 首次看到本屏（首次离线超过 1 小时回来） | 顶部加一行小字「这是你离线时门派的故事。点击「一并领下」可一键收齐奖励。」3 次后不再显示 |
| 第一次有"领悟"类事件 | 该条文字旁边出现轻微金色"·"，提示玩家点进去看 |

## 4. A 级界面（核心循环必经）

### A1 · 主菜单

**解锁阶段**：永久（启动后第 1 屏）
**层级**：根

**核心信息**：
- 游戏 logo（写实武侠风，水墨）
- 版本号（小字）
- 4 个按钮

**主要操作**：
- **新游戏**：开新档（覆盖当前 slot 前要二次确认）
- **继续**：直接进 A3 上线第一屏（用最近的 slot）
- **设置**：跳 C6
- **退出**：仅 Windows，关进程

**入口/出口**：
- 来自：游戏启动
- 去往：A2 存档选择 / A3 上线第一屏 / C6 设置

**新手引导锚点**：无（界面自解释）

**Phase 1 简化**：本阶段先用 D9 调试菜单替代主菜单，Phase 3 再实现正式版

---

### A2 · 存档选择（Phase 5 解锁）

**解锁阶段**：Phase 5 多存档功能完成时
**层级**：A1 → A2

**核心信息**：
- 3 个槽位卡片，每个显示：
  - slotName（玩家自定义名）/ 空槽显示「空着」
  - sectName（门派名）
  - 祖师名 + 当前境界
  - lastSavedAt（X 天前）
  - 累计游戏时间
  - 历史最高爬塔层

**主要操作**：
- 点击非空槽位 → A3 上线第一屏
- 点击空槽位 → 新建存档流程（祖师起名、属性 roll、门派起名）
- 长按非空槽位 → 二次确认后删除（C6 → 存档管理也能进入此操作）

**入口/出口**：
- 来自：A1 主菜单
- 去往：A3 上线第一屏 / 新建角色流程

**新手引导锚点**：第一次进入显示「最多保存 3 份江湖路。每份独立，互不干涉。」

## 10. 视觉与交互约定

### 10.1 字体

| 用途 | 字体 | 替代 |
|------|------|------|
| 大标题（界面名、境界名） | 思源宋体 Heavy | 系统衬线字体 |
| 正文 | 思源宋体 Regular | 系统衬线字体 |
| 数值（伤害、HP、属性） | 等宽数字（思源宋体的等宽数字版本） | 系统等宽字体 |
| 战斗飘字 | 思源宋体 Bold + 描边 | 加粗系统衬线字体 |

**禁用**：圆体、卡通体、艺术变形字体（破坏写实武侠基调）

### 10.2 配色

| 用途 | 主色 | 辅助 |
|------|------|------|
| 背景 | 暗宣纸 `#1a1612` | 渐变到 `#2a2218` |
| 文字 | 浅米白 `#e8dfc7` | 重要金 `#c9a961` |
| 流派 - 刚猛 | 暗红 `#a53e3e` | 击中震波 `#d97070` |
| 流派 - 灵巧 | 暗金 `#c9a961` | 暴击金光 `#f0d68a` |
| 流派 - 阴柔 | 暗紫 `#5d3a6d` | 雾气 `#9070a8` |
| HP 高（>50%） | 沉绿 `#5d8a4f` | — |
| HP 中（25-50%） | 暗黄 `#b09040` | — |
| HP 低（<25%） | 红 `#a53e3e` | 闪烁警示 |

**禁用**：高饱和度 RGB 原色（红 #ff0000、绿 #00ff00、蓝 #0000ff）—— 都太刺眼

### 10.3 动画时长约定

| 类型 | 时长 |
|------|------|
| 界面切换淡入淡出 | 200-300ms |
| 战斗攻击动画 | 400ms（前冲 150 + 出招停 100 + 后撤 150） |
| 伤害飘字 | 800ms（向上漂浮 + 淡出） |
| 普通弹窗淡入 | 200ms |
| 庆祝弹窗（D4-D8） | 2-3 秒（首次） / 1 秒（之后） |
| 上线第一屏文字渐入 | 每段 200ms 延迟 |

### 10.4 反馈强度梯度

| 强度 | 用途 |
|------|------|
| 极轻 | 普通操作（按钮按下、切 tab）：仅颜色微变化，无音效 |
| 轻 | 信息更新（HP 减少、内力增加）：数字滚动 100-200ms |
| 中 | 战斗事件（命中、闪避）：飘字 + 短促音效 |
| 强 | 关键时刻（暴击、流派克制、突破）：屏震 + 金光 + 较长音效 |
| 极强 | 首次解锁系统、首次得宝、领悟新招：全屏特效 + 暂停游戏让玩家欣赏 |

**原则**：日常操作越轻越好，关键时刻越浓越好，让"惊喜"真的是惊喜。
